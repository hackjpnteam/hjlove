🔧 Google OAuth設定確認ガイド

現在のエラー: "The given origin is not allowed for the given client ID"
これはGoogle Cloud Consoleでの承認済みオリジン設定が必要であることを示しています。

📍 現在の環境情報

あなたのオリジン:

クライアントID: 770946554968-cf14im7fgm9ng8lmko4ltik2rns2udhb.apps.googleusercontent.com

✅ ステップ1: Google Cloud Consoleにアクセス

  1. Google Cloud Console - 認証情報ページを開く
  2. 正しいプロジェクトが選択されていることを確認（上部のプロジェクト名を確認）
  3. 「OAuth 2.0 クライアント ID」セクションを見つける
  4. クライアントID 770946554968-cf14im7fgm9ng8lmko4ltik2rns2udhb... をクリック

✅ ステップ2: 承認済みのJavaScript生成元を追加

「承認済みの JavaScript 生成元」セクションに以下のURLをすべて追加してください：

http://localhost:8000 http://localhost:3000 http://localhost:3001 http://127.0.0.1:8000 http://127.0.0.1:3000 http://127.0.0.1:3001
注意: 各URLは別々の行に追加してください。「URIを追加」ボタンをクリックして1つずつ追加します。

✅ ステップ3: リダイレクトURIを追加（オプション）

「承認済みのリダイレクト URI」セクションに以下を追加：

http://localhost:8000/auth-callback.html http://localhost:3000/auth-callback.html http://localhost:3001/auth-callback.html

✅ ステップ4: 保存して待機

  1. 画面下部の「保存」ボタンをクリック
  2. 5-10分待つ（Google側で設定が反映されるまで時間がかかります）
  3. ブラウザのキャッシュをクリア（Cmd+Shift+R または Ctrl+Shift+R）

🧪 ステップ5: テスト

設定が反映されたら、以下のリンクでテストしてください：

  • シンプルテストページ
  • ログインページ
  • 詳細診断ページ

❓ よくある問題と解決策

問題: 設定を保存したのにまだエラーが出る

  • Google側の反映に5-10分かかります。しばらく待ってください。
  • ブラウザを完全に閉じて再起動してみてください。
  • プライベート/シークレットモードで試してみてください。

問題: "FedCM migration" の警告が出る

これは将来的な変更に関する警告で、現在の動作には影響しません。無視して構いません。

設定が完了したら

Google認証ボタンをクリックすると、Googleのログイン画面が表示されるはずです。
